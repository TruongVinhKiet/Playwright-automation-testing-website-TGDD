Playwright là gì?

Định nghĩa: Framework mã nguồn mở của Microsoft (2020).
Mục tiêu: Kiểm thử End-to-End (E2E) cho Web hiện đại.
Đa trình duyệt: Hỗ trợ hoàn hảo Chromium (Chrome, Edge), WebKit (Safari), Firefox.
Đa nền tảng: Hoạt động nhất quán trên Windows, Linux, macOS.

Tại sao chọn Playwright?

Các tính năng cốt lõi tạo nên sự khác biệt

Auto-wait

Tự động chờ (Auto-wait) element sẵn sàng (hiển thị, có thể click) trước khi hành động.

→ Loại bỏ lỗi Flaky

Web-first Assertions

Cơ chế kiểm tra cực kỳ thông minh, có khả năng tự động thử lại (retry) các assertion liên tục cho đến khi đạt điều kiện.

Tốc độ cực nhanh

Sử dụng giao thức WebSocket trực tiếp thay vì giao tiếp chậm chạp qua HTTP.

→ Tương tác tức thì

Kiến trúc 3 tầng cốt lõi

1. Browser

Trình duyệt vật lý (Chromium, WebKit, Firefox). Cho phép chạy đa instance.

2. Browser Context

Môi trường cô lập (như "Tab ẩn danh"). Tách biệt Cookie → Test song song an toàn.

3. Page

Đại diện cho 1 Tab/Cửa sổ. Nơi trực tiếp thực hiện mọi thao tác UI.

Các thành phần chính trong Test Script

Locator

Giúp tìm kiếm và định vị chính xác đối tượng DOM trên trang web.

page.locator('.btn')

Action

Hành động tương tác để mô phỏng người dùng thật sự thao tác.

.click(), .fill(), .type()

Assertion

Hàm xác nhận kết quả tự động hóa để đảm bảo test chuẩn.

expect(page).toHaveTitle()

Khả năng kỹ thuật linh hoạt khác

Network Interception

Can thiệp trực tiếp, lắng nghe và chỉnh sửa request/response mạng dễ dàng.

API Testing

Hỗ trợ gọi và kiểm thử API (REST) nhanh chóng ngay trong nội hàm script.

Mobile Emulation

Giả lập chuẩn xác thiết bị di động (kích thước, GPS, permissions, User-Agent).

Hệ sinh thái công cụ (Tooling)

Codegen

Ghi lại thao tác người dùng (Record) và tự động sinh ra mã code script (nhiều ngôn ngữ).

Trace Viewer

Hộp đen ghi lại toàn bộ tiến trình chạy (Video, Screenshots, Network Log) cực đỉnh để fix bug.

UI Mode & VS Code

Tích hợp cực sâu bào trình soạn thảo, giao diện đồ họa đẹp, hỗ trợ debug Step-by-step.
